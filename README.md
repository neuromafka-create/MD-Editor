# MD Editor

**MD Editor** — Markdown-редактор для десктопа и браузера (v1.2.1).

Стек: **Vite 8**, **Tauri 2**, vanilla JS, **marked**, **highlight.js**, **docx**.

Редактор с живым HTML-предпросмотром, вкладками, оглавлением, поиском/заменой, меню «Файл», форматированием, таблицами, экспортом в HTML/DOCX и нативной работой с файлами в десктоп-сборке.

## Возможности

### Редактор и предпросмотр

- Двухпанельный интерфейс: исходный Markdown и HTML-предпросмотр
- Режимы отображения: обе панели / только редактор / только превью
- Полный экран (`F11`)
- Изменяемая граница между редактором и предпросмотром (перетаскивание, клавиатура, двойной щелчок — 50/50)
- Живой рендер через [marked](https://github.com/markedjs/marked)
- Нумерация строк и подсветка текущей строки
- Статус-бар: число слов, символов, позиция курсора (строка / колонка)
- Копирование содержимого блоков кода в превью (кнопка **Copy**)
- **Подсветка синтаксиса** в блоках кода (highlight.js, тема github-dark)
- **Метка языка** в левом верхнем углу блоков кода (bash, python, js и др.)
- Тёмная и светлая темы (сохраняются в `localStorage`)
- **Автосохранение** каждые 3 секунды + восстановление сессии после перезагрузки страницы

### Вкладки

- Несколько документов одновременно
- **Создать** / **Открыть** — открывают отдельную вкладку
- Переключение вкладок с сохранением содержимого, курсора и прокрутки
- **Ctrl+Tab** — быстрое переключение по последнему использованию (MRU)
- Отдельная история undo/redo на каждую вкладку
- Индикатор несохранённых изменений (точка на вкладке)
- Закрытие вкладки с подтверждением, если есть несохранённые правки

### Файлы и меню

Меню **Файл ▾** и иконки на панели инструментов:

| Действие | Поведение |
|----------|-----------|
| **Создать** (`Ctrl+N` / `Ctrl+T`) | Новая вкладка |
| **Открыть** | Диалог выбора `.md` / `.markdown` / `.mdown` / `.txt` |
| **Сохранить** (`Ctrl+S`) | Тихая перезапись уже открытого файла (без диалога) |
| **Сохранить как…** (`Ctrl+Shift+S`) | Диалог выбора пути / имени |
| **Экспорт HTML** | Сохранение как HTML-документ |
| **Экспорт DOCX** | Сохранение как Word-документ |
| Новый документ → Сохранить | Сначала «Сохранить как…», далее — тихая запись |

- **Tauri (десктоп):** нативные диалоги и запись на диск (`src-tauri`, rfd)
- **Браузер (Chrome / Edge):** [File System Access API](https://developer.mozilla.org/en-US/docs/Web/API/File_System_Access_API) — handle файла для повторного сохранения
- **Fallback:** download, если нет привязки к файлу или API недоступен

### Таблицы

- GFM-таблицы в Markdown рендерятся в превью
- Кнопка **Copy** у каждой таблицы: копирует **HTML + TSV** (удобно вставлять в Word и Excel)
- Экспорт в **DOCX** с настоящими таблицами Word (`Table` / `TableRow` / `TableCell`)

Пример:

```markdown
| Колонка A | Колонка B |
|-----------|-----------|
| Значение  | Значение  |
```

### Экспорт

- **HTML** — полноценный HTML-документ
- **DOCX** — Word-документ (библиотека `docx`): заголовки, списки, код, цитаты, ссылки, таблицы, верхний/нижний индекс и др.
- **PDF** — A4, через `html2pdf.js` (HTML-превью → PDF); в десктопе локальные картинки встраиваются как data URL

### Форматирование

Панель инструментов (SVG-иконки, сгруппирована по блокам):

- Отмена / повтор
- Вырезать / копировать / вставить
- **Жирный**, *курсив*, ~~зачёркивание~~, `инлайн-код`, ==выделение==
- Верхний индекс (`<sup>`) и нижний индекс (`<sub>`)
- Заголовки H1–H4
- Маркированный и нумерованный списки, списки задач (`- [ ]`)
- Цитата, блок кода, горизонтальная линия
- Вставка изображения (data URL) и ссылки
- Автозакрытие скобок и кавычек: `()`, `[]`, `""`, `''`, `` ` ``

**Подсветка активных кнопок:** если курсор находится внутри соответствующего форматирования (например, внутри `# Заголовок`), кнопка на панели подсвечивается.

**Тогл форматирования:** повторное нажатие кнопки снимает форматирование (ищет ближайшие маркеры вокруг выделения).

### Контекстное меню

Выпадающее меню по правому клику на выделенном тексте:

| Группа | Пункты |
|--------|--------|
| Inline | Жирный, Курсив, Зачёркивание, Подсветка, Инлайн-код |
| Индексы | Верхний индекс, Нижний индекс |
| Списки | Маркированный, Нумерованный, Задач |
| Блоки | Цитата, Блок кода |
| Медиа | Ссылка, Изображение |

### Оглавление (TOC)

- Боковая панель структуры документа (H1–H4)
- Клик по пункту — переход к заголовку в редакторе и в превью
- Подсветка текущего раздела по позиции курсора
- Показ / скрытие панели (кнопка **Оглавление**, состояние в `localStorage`)
- **Вставить** — вставить блок TOC в документ (или обновить, если уже есть)
- **Обновить** — пересобрать существующий блок TOC

Блок в Markdown:

```markdown
<!-- TOC -->
## Оглавление

- [MD Editor](#md-editor)
  - [Возможности](#возможности)
  - [Стек](#стек)
  - [Структура проекта](#структура-проекта)
  - [Требования](#требования)
  - [Установка](#установка)
  - [Запуск](#запуск)
  - [Сборка](#сборка)
  - [Скрипты npm](#скрипты-npm)
  - [Заметки](#заметки)
  - [Лицензия](#лицензия)

<!-- /TOC -->
```

В превью у заголовков выставляются якоря (`id`), ссылки из оглавления прокручивают к нужному месту.

### Поиск и замена

- Найти далее (циклически)
- Заменить текущее вхождение
- Заменить все
- Счётчик совпадений `N / M`

### Горячие клавиши

Работают по `event.code` (не зависят от раскладки клавиатуры).  
На macOS вместо `Ctrl` используется `Cmd`.

| Клавиши | Действие |
|---------|----------|
| `Ctrl+B` | Жирный |
| `Ctrl+I` | Курсив |
| `Ctrl+Shift+X` | Зачёркивание |
| `Ctrl+Shift+H` | Подсветка |
| `Alt+1`..`Alt+4` | Заголовок H1–H4 |
| `Ctrl+S` | Сохранить |
| `Ctrl+Shift+S` | Сохранить как… |
| `Ctrl+N` | Новая вкладка |
| `Ctrl+T` | Новая вкладка |
| `Ctrl+W` | Закрыть вкладку |
| `Ctrl+Tab` | Следующая вкладка (MRU) |
| `Ctrl+Shift+Tab` | Предыдущая вкладка (MRU) |
| `Ctrl+F` | Фокус в поле поиска |
| `Ctrl+H` | Фокус в поле замены |
| `F11` | Полный экран |
| `Ctrl+Z` | Отменить |
| `Ctrl+Y` / `Ctrl+Shift+Z` | Повторить |

В браузере также перехватываются `Ctrl+N` и `Ctrl+Shift+T`, чтобы не открывать новые окна/вкладки браузера.

## Запуск

### Веб (браузер)

```bash
npm run dev
```

Откройте в браузере адрес из терминала, обычно:

```
http://localhost:5173
```

Для полноценного «Сохранить без диалога» удобнее **Chrome** или **Edge** (File System Access API).

### Десктоп (Tauri)

```bash
npm run tauri:dev
```

или:

```bash
npm run tauri dev
```

Откроется окно приложения с нативными диалогами файлов.

## Сборка

### Веб (статика в `dist/`)

```bash
npm run build
```

Сборка включает `index.html` и `about.html`. Каталог `dist/` готов к деплою (в т.ч. на Netlify).

### Десктоп (Tauri) / установщик Windows

```bash
npm run tauri:build
```

Скрипт сначала синхронизирует версию из `package.json` в `tauri.conf.json`, затем собирает release.

Сборка создаёт:

| Артефакт | Путь |
|----------|------|
| **Установщик NSIS (x64)** | `src-tauri/target/release/bundle/nsis/MD Editor_1.2.1_x64-setup.exe` |
| Исполняемый файл | `src-tauri/target/release/md-editor.exe` |

Установщик:

- ставит приложение для текущего пользователя или для всех (выбор в мастере);
- языки мастера: English / Russian;
- добавляет ярлык в меню «Пуск» (папка **MD Editor**);
- при необходимости скачивает bootstrapper **WebView2** (нужен для интерфейса).

Запуск установщика: двойной щелчок по `MD Editor_1.2.1_x64-setup.exe`.

## Заметки

- Верхний и нижний индекс вставляются как HTML-теги `<sup>` / `<sub>` — они корректно отображаются в превью и учитываются при экспорте в DOCX.
- Подсветка синтаксиса работает для популярных языков (bash, python, js, ts, html, css, json, go, rust и др.).
- Таблицы копируются в буфер как HTML (для Word) и как TSV (для Excel и простого текста).
- Изображения: кнопка на панели вставляет **data URL**; короткие/относительные пути в `![](photo.png)` в **десктопе** резолвятся относительно каталога открытого `.md` (через Tauri asset protocol). Документ нужно открыть с диска (`Файл → Открыть`), иначе относительный путь некуда привязать.
- Изображения-data URL удобны для автономных документов, но сильно увеличивают размер файла.
- Горячие клавиши завязаны на физические клавиши (`KeyB`, `KeyS` и т.д.), поэтому работают при русской и других раскладках.
- Настройки темы, видимости оглавления, пропорции панелей и режима отображения хранятся в `localStorage`.
- Автосохранение работает каждые 3 секунды (только если есть несохранённые изменения). Сессия восстанавливается автоматически при перезагрузке страницы.
- Контекстное меню (правый клик) доступно только при выделенном тексте.
